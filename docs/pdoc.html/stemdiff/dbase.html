stemdiff.dbase

stemdiff.dbase

Functions for the reading of 4D-STEM datafiles to create database of all files.

The database contains [filename, S-entropy and XY-center] of each datafile.

  • The database enables fast filtering of datafiles and fast access to datafile features.
  • S-entropy = Shannon entropy = a fast-to-calculate image feature; datafiles with high S contain strong diffractions and vice versa.
  • XY-center = two values = X- and Y-coordinate of the central spot (primary beam) for given datafile.

 24def calc_database(DATAFILES,CENTERING):
 25    """
 26    Read 4D-STEM datafiles and calculate database of all files,
 27    which contains [filename, S-entropy and XY-center] for each datafile.
 28    
 29    * The database can be calculated, saved and read back with functions
 30      *calculate_database*, *save_database*, and *read_database*,
 31      respectively (all functions are defined in dbase module).
 32    * This can save time, as the large databases do not have to be
 33      calculated again and again when we fine-tune calculation parameters.
 34    * Within one STEMDIFF/Spyder session, it may not be even necessary
 35      to read the database as it is in memory if we re-use *master script*.
 36    * See *stemdiff.__init__* that shows a typical STEMDIFF *master script*.
 37    
 38    Parameters
 39    ----------
 40    DATAFILES : glob object
 41        Names of all datafiles from 4D-STEM dataset.
 42        The *glob object* is usually defined in the *master script*.
 43        See `stemdiff.__init__` that shows a typical/template *master script*.
 44            
 45    CENTERING : centering object
 46        Object containing parameters for finding center of diffractograms.
 47        The *centering object* is usually defined in the *master script*.
 48        See `stemdiff.__init__` that shows a typical/template *master script*.
 49        
 50    Returns
 51    -------
 52    df : pandas DataFrame object
 53        Database contains [filename, ShannonEntropy, XY-center]
 54        of each datafile in 4D-STEM dataset.
 55    """
 56    # Prepare coordinates of the center
 57    xc,yc = (None,None)
 58    # Prepare empty list
 59    # (appending to lists is more efficient than appending to np/pd-structures
 60    list_of_datafiles = []
 61    # Go through datafiles and calculated their entropy
 62    for datafile in DATAFILES:
 63        # a) Read datafile
 64        arr = stemdiff.io.read_datafile(datafile)
 65        # b) Calculated and print Shannon entropy of the datafile
 66        entropy_value = measure.shannon_entropy(arr)
 67        # c) Calculate center of the datafile
 68        # Note: the center is calculated for enlarged/rescaled array
 69        # (this gives better accuracy for rescaled arrays
 70        # (for original arrays, divide center coordinates by RESCALE
 71        if CENTERING.ctype == 2:
 72            arr = stemdiff.io.rescale_array(arr, RESCALE)
 73            xc,yc = stemdiff.io.find_array_center(
 74                arr, CENTERING.csquare, CENTERING.cintensity)
 75        elif (CENTERING.ctype == 1) and (xc == None):
 76            arr = stemdiff.io.rescale_array(arr, RESCALE)
 77            xc,yc = stemdiff.io.find_array_center(
 78                arr, CENTERING.csquare, CENTERING.cintensity)
 79        elif (CENTERING.ctype == 0) and (xc == None):
 80            xc,yc = (round(DET_SIZE*RESCALE/2),round(DET_SIZE*RESCALE/2))
 81        # d) Append all calculated values to list
 82        list_of_datafiles.append([datafile, entropy_value, xc, yc])
 83    # Convert list to pandas DataFrame
 84    df = pd.DataFrame(list_of_datafiles,
 85                      columns=['DatafileName','Entropy','Xcenter','Ycenter'])
 86    # Return the dataframe containing names of datafiles + their entropies
 87    return(df)

 89def save_database(df, output_file):
 90    """
 91    Save database, which contains [filenames, entropies and XY-centers]
 92    of all 4D-STEM datafiles; the dbase is saved as pickled object/zip-file.
 93    
 94    * The database can be calculated, saved and read back with functions
 95      *calculate_database*, *save_database*, and *read_database*,
 96      respectively (all functions are defined in dbase module).

 98      calculated again and again when we fine-tune calculation parameters.
 99    * Within one STEMDIFF/Spyder session, it may not be even necessary
100      to read the database as it is in memory if we re-use *master script*.
101    * See *stemdiff.__init__* that shows a typical STEMDIFF *master script*.
102    
103    Parameters
104    ----------
105    df : pandas DataFrame object
106        This object is a database of all datafiles,
107        which contains [filenames, entropies and XY-centers]
108        of each datafile in 4D-STEM dataset.
109    output_file : str
110        Filename of the output file (without extension).
111        The database is saved as a pickled object/zip-file.
112
113    Returns
114    -------
115    None
116        The output is the database *df* saved as *output_file* on a disk.
117    """
118    df.to_pickle(output_file)

120def read_database(input_database):
121    """
122    Read database, which contains [filenames, entropies and centers]
123    of all 4D-STEM datafiles.
124    
125    * The database can be calculated, saved and read back with functions
126      *calculate_database*, *save_database*, and *read_database*,
127      respectively (all functions are defined in dbase module).
128    * This can save time, as the large databases do not have to be
129      calculated again and again when we fine-tune calculation parameters.
130    * Within one STEMDIFF/Spyder session, it may not be even necessary
131      to re-read the database if we re-run *master script*
132      (because df object remains in memory).
133    * See *stemdiff.__init__* that shows a typical STEMDIFF *master script*.
134
135    Parameters
136    ----------
137    input_database : str or pandas.DataFrame
138        * str = filename of the input file that contains the database.
139        * pandas.Dataframe = dataframe that contains the database.
140        * Why two possile types of input?
141          If the database is in memory in the form of pandas.DataFrame
142          (which is quite common), it is useles to re-read it from file.
143          
144    Returns
145    -------
146    df : pandas DataFrame object
147        Database that has been read from disk or pandas.Dataframe.
148    """
149    # Read database from [input_file]
150    # NOTE: input_database is either saved/pickled file or pd.DataFrame
151    # Reason: sometimes it is more efficient to use pd.DataFrame directly 
152    if type(input_database) == pd.DataFrame:
153        df = input_database
154    else:
155        df = pd.read_pickle(input_database)
156    return(df)

182def get_high_S_files(dbase, S=None, P=None, N=None):
183    """
184    Get filenames and parameters of high-entropy datafiles
185    from a database, which contains [filenames, entropies and centers]
186    of all 4D-STEM datafiles. 
187
188    Parameters
189    ----------
190    dbase : str
191        Filename of the database file
192        = zip-file containg pickled database object.    
193    S : float
194        Shannon entropy value;
195        if S is given, we get only the files with entropy > S.
196    P : float
197        Percent of files with the lowest entropy;
198        if P is given, we get only P% of files with the highest entropy.
199    N : integer
200        Number of files with the lowest entropy;
201        if N is given, we get only N files with the highest entropy.
202
203    Returns
204    -------
205    DataFrame iterator
206        The complete database/DataFrame is huge => return DataFrame iterator;     
207        the iterator gradually returns all items/datafiles with high entropy.
208        
209    Notes
210    -----
211    * Priority of parameters: S > P > N.
212    * Therefore, if S is given, P and N are ignored etc.
213    """
214    # Read database file into pandas.DataFrame
215    df = read_database(dbase)
216    # Calculate entropy limit (then we can get files with higher entropy)
217    entropy_limit = get_entropy_limit(df,S,P,N,high_entropy_files=True)
218    # Calculate and return reduced database containing only high-entropy files
219    df2 = df[df.Entropy >= entropy_limit]
220    # Return adjusted/filtered database as DataFrame iterator
221    # (whole DataFrame is huge and not iterable directly
222    return(df2.iterrows())

266def get_entropy_limit(df,S,P,N,high_entropy_files=True):
267    """
268    Determine Shannon entropy limit;
269    which separates high- and low-entropy files;
270    (the entropy limit can be determined from S or P or N parameter).
271
272    Parameters
273    ----------
274    df : pandas DataFrame object
275        This object is a database of all 4D-STEM datafiles,
276        which contains [filenames, entropies and centers] of diffractograms.
277    S : Shannon entropy
278        that separaters high- and low-S files;
279        (trivial case: the function just returns the value of S).
280    P : Percent of files
281        that determines how many percent of high- or low-S files we want;
282        (here we calculate the S-value separating the high/low-S files and
283        the calculated value depends on parameter high_entropy_files below).
284    N : Number of files
285        that determines how many high- or low-S files we want;
286        (here we calculate the S-value separating the high/low-S files and
287        the calculated value depends on parameter high_entropy_files below).
288    high_entropy_files : boolean, optional, default=True
289        * If the parameter is True, the S-value separates high-entropy files.
290        * If the parameter is False, the S-value separates low-entropy files.
291        * Example: if we have P=20, we can separate either
292          20% of the high-S files (if high_entropy_files = True)
293          or 20% of the low-S files (if the high_entropy_files = False).
294          
295    Returns
296    -------
297    entropy limit : float
298        The value of Shannon entropy,
299        that separates high- and low-entropy files.
300        
301    Notes
302    -----
303    * Priority of parameters: S > P > N.
304    * Therefore, if S is given, P and N are ignored etc.
305    """
306    # Calculate histogram, pdf and cdf
307    # (we need high precision: bins=1000
308    # (in order to get correct/non-approximate number of files for switch N
309    counts,bins,pdf,cdf = calculate_entropy_histogram(df,bins=1000)
310    # A) Entropy limit is given by S
311    if S:
312        entropy_limit = S
313    # B) Entropy limit is given by P or N 
314    else:
315        # 1) determine percent of files - this is given by P or N
316        # P = percent of high- or low-entropy files
317        # N = number of  high- or low-entropy files
318        if P: percent = P
319        else: percent = N / np.sum(counts) * 100
320        # 2) if we want high-entropy files, percent = 100-percent
321        if high_entropy_files: percent = 100-percent 
322        # 3) Now that we have percent of files, we can calculate entropy limit
323        cdf_limit = percent/100
324        entropy_limit = bins[np.argmax(cdf>=cdf_limit)]
325    return(entropy_limit)

327def calculate_entropy_histogram(df, bins):
328    """
329    Calculate parameters,
330    which can be used for plotting the entropy histogram.
331
332    Parameters
333    ----------
334    df : pandas DataFrame object
335        This object is a database of (pre-selected) 4D-STEM datafiles,
336        which contains [filenames, entropies and centers] of diffractograms.
337    bins : int
338        Number of bins ~ intervals of the histogram.
339
340    Returns
341    -------
342    counts, bins, pdf, cdf
343        * counts = array; number of files in bins
344        * bins = array containing bins ~ intervals ~ limits between values
345        * pdf = probability distribution function = counts normalized to 1
346        * cdf = cummulative distribution function - calculated by np.cumsum
347    """
348    counts, bins = np.histogram(df.Entropy, bins=bins)
349    pdf = counts / sum(counts)
350    cdf = np.cumsum(pdf)
351    return(counts,bins,pdf,cdf)

353def plot_entropy_histogram(df, bins):
354    """
355    Create plot/histogram of entropy values of all datafiles
356    and show it on the screen.
357
358    Parameters
359    ----------
360    df : pandas DataFrame object
361        Database containing all datafiles;
362        df object is obtained as the output from the function calc_database.
363        
364    bins : integer
365        Number of bins = intervals in the histogram; typical value is 100.
366
367    Returns
368    -------
369    None.
370        The result is the entropy histogram on the screen.
371    """
372    counts,bins,pdf,cdf = calculate_entropy_histogram(df, bins)
373    plt.plot(bins[1:],pdf, 'b-', label='PDF')
374    plt.plot(bins[1:],cdf, 'r-', label='CDF')
375    plt.title('Shannon entropy distribution')
376    plt.legend()
377    plt.grid()
378    plt.show()

328def calculate_entropy_histogram(df, bins):
329    """
330    Calculate parameters,
331    which can be used for plotting the entropy histogram.
332
333    Parameters
334    ----------
335    df : pandas DataFrame object
336        This object is a database of (pre-selected) 4D-STEM datafiles,
337        which contains [filenames, entropies and centers] of diffractograms.
338    bins : int
339        Number of bins ~ intervals of the histogram.
340
341    Returns
342    -------
343    counts, bins, pdf, cdf
344        * counts = array; number of files in bins
345        * bins = array containing bins ~ intervals ~ limits between values
346        * pdf = probability distribution function = counts normalized to 1
347        * cdf = cummulative distribution function - calculated by np.cumsum
348    """
349    counts, bins = np.histogram(df.Entropy, bins=bins)
350    pdf = counts / sum(counts)
351    cdf = np.cumsum(pdf)
352    return(counts,bins,pdf,cdf)
